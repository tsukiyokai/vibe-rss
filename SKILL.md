# vibe-rss阅读周报生成器

## 工作流程

```
sources.txt --> fetch_urls.py --> Claude筛选+摘要 --> weekly-YYYY-WNN.md --> vibe-fmt
```

## 步骤

### 1. 读取sources.txt

读取 `~/.claude/skills/vibe-rss/sources.txt`，解析：
- `>` 前缀行 → 兴趣标签列表
- `#` 行 → 分类（忽略）
- 裸URL → 待抓取源列表

### 2. 抓取内容

运行抓取脚本：

```bash
python3 ~/.claude/skills/vibe-rss/scripts/fetch_urls.py ~/.claude/skills/vibe-rss/sources.txt
```

脚本输出JSON，每个元素包含：
- url: 源URL
- title: 页面标题
- articles: 该源最近的文章列表（标题+链接+摘要片段）
- error: 抓取失败时的错误信息

HN专用字段（由Firebase API提供）：
- articles[].score: 投票分数
- articles[].num_comments: 评论数
- articles[].hn_link: HN讨论页链接
- articles[].top_comments: score前10篇文章的精选评论(每篇最多5条)

### 3. 筛选文章

根据兴趣标签和AI判断，从抓取结果中筛选值得推荐的文章。

筛选标准（优先级从高到低）：
1. 与兴趣标签的匹配度
2. 文章的深度和原创性（优先技术深度文、经验总结、独到见解）
3. 时效性（优先最近一周发布）
4. 实用性（能学到新东西或改变认知）

不设数量上限。宁缺毋滥，但好文章不遗漏。

HN特殊处理：
- score >= 200 且评论数 >= 30 的HN文章自动入选，即使不严格匹配兴趣标签（高讨论度本身说明值得关注）
- 入选的HN文章保留精选评论，在周报中展示。精选评论优先保留提供现实经验的：生产环境踩坑、实际项目应用、具体数据或反例，而非纯观点或玩笑
- 当同一文章同时出现在个人博客/专业源和HN时，合并为一条：用专业源的深度分析作为摘要，同时附上HN讨论链接和精选评论

### 4. 并行获取全文并生成摘要

将筛选出的文章按主题聚类后分组，启动多个subagent并行处理。

分组策略：按兴趣标签做简单聚类，确保相关文章在同一组。例如：
- 同一事件的多角度报道（如供应链攻击 + 依赖安全评论）归为一组
- 同一技术领域（如编译器相关的多篇）归为一组
- 无明显关联的文章可自由分配以平衡组大小

每个subagent的任务：
1. 对分配的文章逐篇WebFetch获取全文
2. 按步骤5的格式生成该组文章的周报条目（含中文标题、推荐理由、摘要）
3. HN文章需保留精选评论

主agent收集所有subagent的输出后，合并排序生成最终周报。

### 5. 生成周报

输出文件：`./report/weekly-YYYY-WNN.md`

格式：

```markdown
# 阅读周报 YYYY-WNN

> 本周从 N 个源中筛选出 M 篇推荐文章。

## 1. 中文标题
- 原文标题: Original Title Here
- 来源: [example.com](https://example.com/article)
- 阅读时间/难度: 15 min / 中级
- 推荐理由: 一句话点明为什么值得读——改变什么认知、解决什么问题、或揭示什么趋势
- 摘要: 3-5句话提炼核心论点和关键洞察，重点是作者的论证逻辑而非复述开头

## 2. ...

---

以下来自 Hacker News 热门讨论，附精选评论。

## N. 中文标题
- 原文标题: Original Title Here
- 来源: [example.com](https://example.com/article) | [HN讨论](https://news.ycombinator.com/item?id=xxx) (550 pts, 326 comments)
- 阅读时间/难度: 10 min / 初级
- 推荐理由: ...
- 摘要: ...
- HN精选评论:
  - "评论1摘要"
  - "评论2摘要"

---

## 本周观察

跨文章的趋势、张力或值得留意的信号，3-5条，每条1-2句。

- ...
- ...
```

生成"本周观察"时，识别文章之间的共振（多篇指向同一趋势）和张力（同一话题的不同立场），提炼为简短的判断句。

### 6. 格式化

对生成的周报文件执行 `/vibe-fmt`。

## 约束

- 全中文输出（原文标题保留原文）
- 难度分三级：初级、中级、高级，与阅读时间合并为一行
- 来源用markdown链接格式，域名为文本、完整URL为链接
- 推荐理由只回答"so what"：对读者意味着什么，不复述文章内容
- 摘要要提炼核心观点，不是简单翻译开头段落
- 每篇文章独立成文，不写"与上一篇互为补充"等关联说明
